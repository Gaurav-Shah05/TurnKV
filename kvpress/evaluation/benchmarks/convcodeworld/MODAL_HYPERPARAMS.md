# ConvCodeWorld Modal Hyperparameters

Source of truth: `modal_app.py`. Invoke from the `kvpress/` directory:

```bash
modal run evaluation/benchmarks/convcodeworld/modal_app.py::main [flags]
```

Modal exposes Python `snake_case` parameters as CLI flags with hyphens. For
example, `compression_ratio` is passed as `--compression-ratio`.

## Dispatch Parameters

These parameters are handled by the local Modal entrypoint before each remote
worker run starts.

| Flag | Type | Default | Meaning |
|---|---:|---|---|
| `--benchmark-mode` | `str` | `live` | Benchmark execution mode. `live` feeds generated-code feedback into later turns; `static` teacher-forces reference code/feedback from ConvCodeWorld while still executing each generated attempt. Aliases: `live-loop`, `static-replay`. |
| `--press-names` | `str` | `snapkv,streaming_llm,expected_attention` | Comma-separated list of presses. The entrypoint launches one remote run per press and passes each value as `press_name`. |
| `--detach-remote` / `--no-detach-remote` | `bool` | `False` | If true, spawn remote jobs and print Modal object IDs/dashboard URLs instead of waiting for each job. |

## Model And Runtime Parameters

| Flag | Type | Default | Meaning |
|---|---:|---|---|
| `--model` | `str` | `deepseek-ai/DeepSeek-R1-Distill-Llama-8B` | Code-generation model. Also stored in `KV_PRESS_CONVCODEWORLD_MODEL` inside Modal. |
| `--feedback-model` | `str` or `None` | `google/gemma-4-E2B-it` | Model used for simulated verbal feedback. If unset in `live_loop.py`, the code model is reused. |
| `--attn-implementation` | `str` or `None` | `flash_attention_3` | Attention backend for the code model. Use `eager` only for debugging fallback. |
| `--feedback-attn-implementation` | `str` or `None` | `flash_attention_3` | Attention backend for the feedback model. |
| `--cot` / `--no-cot` | `bool` | `True` | Adds a brief "think through the feedback" instruction to the initial coding context. |
| `--network-isolation` | `str` | `auto` | Passed to the code executor. `auto` tries network namespace isolation and falls back if unavailable. |
| `--early-stop-on-pass` / `--no-early-stop-on-pass` | `bool` | `True` | Stop a task's live loop once generated code passes; remaining turns are synthetic metric-excluded rows. |

## Benchmark Sampling Parameters

| Flag | Type | Default | Meaning |
|---|---:|---|---|
| `--feedback-config` | `str` | `CF_EF_UNIT_SNF` | ConvCodeWorld feedback mix. Supported dataset configs include `CF_EF_UNIT_SNF`, `CF_EF_FULL_SNF`, `CF_SEF`, `CF_EF_UNIT_SEF`, and `CF_EF_FULL_SEF`. |
| `--num-eval-examples` | `int` | `1` | Number of tasks to run after filtering/sampling. Use `-1` to run all selected rows. |
| `--fraction` | `float` | `1.0` | Fraction of the filtered BigCodeBench rows to sample before `num_eval_examples` is applied. |
| `--max-turns` | `int` | `10` | Maximum refinement turns per task. ConvCodeWorld trajectories are 10 turns. |

## Generation And Budget Parameters

| Flag | Type | Default | Meaning |
|---|---:|---|---|
| `--max-new-tokens` | `int` | `1024` | Maximum code tokens generated per refinement turn. Decoding is greedy. |
| `--verbal-feedback-max-new-tokens` | `int` | `256` | Maximum tokens generated by the feedback simulator. |
| `--global-budget` | `int` | `4500` | Cache length threshold for turn-boundary global compression. Compression is attempted only when the cache exceeds this value. |
| `--compression-ratio` | `float` | `0.5` | Fraction of KV positions to evict for scorer presses. In the live-loop global path, the target after compression is `global_budget * (1 - compression_ratio)`. |
| `--local-budget` | `int` | `4096` | Target size for answer-suffix decode compression. This limits assistant-generated tokens, not the prefix context. |
| `--decode-compression-interval` | `int` | `128` | Number of single-token decode forwards between answer-suffix compression attempts. Must be positive. |
| `--decode-hidden-states-buffer-size` | `int` | `256` | Recent decode hidden states retained for scorer queries during answer-suffix compression. Set `0` to avoid retaining a tail buffer. |

## Base-Press Parameters

`None` means the Modal runner does not override the press object's registry
default. Most live-loop runs should use `snapkv`, `streaming_llm`,
`expected_attention`, `knorm`, a `turnkv_*` variant, a `baseline_*` variant, or
`no_press`; the current global-compression path requires a `ScorerPress`
compatible base.

| Flag | Type | Default | Applies To | Meaning |
|---|---:|---|---|---|
| `--key-channel-compression-ratio` | `float` or `None` | `None` | `think`, `snap_think` | Fraction of key channels to prune for ThinK. Required when the selected press contains `ThinKPress`. |
| `--threshold` | `float` or `None` | `None` | `DMSPress` wrappers | Score threshold for Dynamic Memory Sparsification. Required for DMS-based presses, though non-scorer DMS wrappers are not accepted by the current ConvCodeWorld global path. |
| `--snapkv-window-size` | `int` or `None` | `None` | `snapkv`, `pyramidkv`, compatible composites | Recent-token observation window for SnapKV-style scoring. Registry default is `64` for `SnapKVPress`. |
| `--snapkv-kernel-size` | `int` or `None` | `None` | `snapkv`, `pyramidkv`, compatible composites | Average-pooling kernel size for smoothing SnapKV attention scores. Registry default is `5` for `SnapKVPress`. |
| `--streaming-llm-n-sink` | `int` or `None` | `None` | `streaming_llm` | Number of initial sink tokens preserved by StreamingLLM. Registry default is `4`. |
| `--expected-attention-n-future-positions` | `int` or `None` | `None` | `expected_attention` | Number of future positions used when estimating expected attention. Registry default is `512`. |
| `--expected-attention-n-sink` | `int` or `None` | `None` | `expected_attention` | Number of initial sink tokens excluded from expected-attention pruning. Registry default is `4`. |
| `--expected-attention-use-covariance` / `--no-expected-attention-use-covariance` | `bool` or `None` | `None` | `expected_attention` | Whether to include query covariance in expected-attention scoring. Registry default is `True`. |
| `--expected-attention-use-vnorm` / `--no-expected-attention-use-vnorm` | `bool` or `None` | `None` | `expected_attention` | Whether to multiply expected-attention scores by value-vector norms. Registry default is `True`. |
| `--expected-attention-epsilon` | `float` or `None` | `None` | `expected_attention` | Additive epsilon before value-norm scaling. If this override is omitted, the Modal `expected_attention` path constructs `ExpectedAttentionPress(epsilon=0.01)`. |

## Turn-Aware Parameters

Passing any policy-specific value can create the missing turn-aware policy when
the selected base press is wrapped into `TurnAwareGlobalPress`. Set the
corresponding `alpha_*` above zero for that policy to affect ranking.

| Flag | Type | Default | Meaning |
|---|---:|---|---|
| `--alpha-floor` | `float` or `None` | `None` | Blend coefficient for `TurnFloorPress` weights. |
| `--alpha-anchor` | `float` or `None` | `None` | Blend coefficient for `RoleBoundaryAnchorPress` weights. |
| `--alpha-loyalty` | `float` or `None` | `None` | Blend coefficient for `LoyaltyPress` weights. |
| `--anchor-beta` | `float` or `None` | `None` | Fraction of each role span protected by role-boundary anchoring. Must be in `[0, 1]`; class default is `0.15`. |
| `--floor-gamma` | `float` or `None` | `None` | Per-turn exponential decay for floor reservations. Must be in `(0, 1]`; class default is `0.9`. |
| `--loyalty-top-p` | `float` or `None` | `None` | Fraction of past keys per query counted as loyalty hits. Must be in `(0, 1]`; class default is `0.25`. |
| `--alpha-floor-len` | `float` or `None` | `None` | Length-proportional coefficient for per-turn floor size. Must be non-negative; class default is `0.3`. |
| `--min-floor-tokens` | `int` or `None` | `None` | Hard minimum number of floor tokens before decay. Must be non-negative; class default is `5`. |

## Modal Infrastructure Constants

These are constants in `modal_app.py`, not CLI flags.

| Name | Value | Meaning |
|---|---|---|
| Modal app | `kvpress-convcodeworld-live` | App name registered with Modal. |
| GPU | `H100!` | Exact H100 request for each remote run. |
| Timeout | `86400` seconds | One-day remote function timeout. |
| CUDA base image | `nvidia/cuda:12.8.1-devel-ubuntu24.04` | Base image used to build the worker. |
| Python | `3.11` | Python version added to the CUDA image. |
| Torch | `2.8.0` | Torch pin installed into `/root/kvpress/.venv`. |
| FlashAttention-3 ref | `v2.8.3` | Git tag used to build the reduced H100 FA3 wheel. |
| Transformers ref | `bc4b330451d0e3e33f4ac63593ed9f245227712e` | Upstream commit installed for Gemma4 support. |
| HF cache volume | `kvpress-hf-cache` | Mounted at `/root/.cache/huggingface`. |
| Results volume | `kvpress-convcodeworld-results` | Mounted at `/root/kvpress/evaluation/results_convcodeworld_live_modal`. |

## `modal_run.sh` Presets

`modal_run.sh` currently runs two detached Modal command groups with
`MODAL_HF_SECRET_NAME=hf-secret`.

Shared args:

```bash
--benchmark-mode live
--model deepseek-ai/DeepSeek-R1-Distill-Llama-8B
--feedback-model google/gemma-4-E2B-it
--attn-implementation flash_attention_3
--feedback-attn-implementation flash_attention_3
--feedback-config CF_EF_UNIT_SNF
--compression-ratio 0.5
--snapkv-window-size 64
--snapkv-kernel-size 5
--streaming-llm-n-sink 4
--expected-attention-n-future-positions 512
--expected-attention-n-sink 4
--expected-attention-use-covariance
--expected-attention-use-vnorm
--expected-attention-epsilon 0.01
--fraction 0.1
--num-eval-examples -1
--local-budget 4096
```

The first group adds:

```bash
--press-names snapkv,streaming_llm,expected_attention
```

The second group adds:

```bash
--press-names snapkv
--alpha-floor 1.0
```
